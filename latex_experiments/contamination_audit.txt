## The Problem

The test set (1,455 sentences in experiments/data/splits/shared/test.tsv) contains
BOTH Tatoeba AND structured sentences. Two separate contamination sources:

1. **Structured sentences in test set**: The ablation conditions (module_loo,
   module_size_ctrl, verb, pronoun, minimal_pairs) were created from a larger
   structured dataset (~4,247 sentences) that was NOT properly split against
   the test set. The shared structured_train.tsv (3,462 rows) is clean, but
   the ablation train.tsv files (e.g., FULL = 3,823 rows) contain 361 extra
   structured sentences that also appear in the test set.

2. **Tatoeba sentences in baselines**: The smart-selection baselines (TF-IDF,
   Length, Vocab) were selected from the full 10K Tatoeba pool, which also
   sourced the Tatoeba portion of the test set.

## Root Cause Analysis

The ablation splits in experiments/data/splits/ablations/ were generated from
a DIFFERENT source than experiments/data/splits/shared/structured_train.tsv.

- structured_train.tsv: 3,462 sentences (CLEAN — proper train split)
- structured_train.csv: 3,462 sentences (same, with metadata)
- ablation FULL/train.tsv: 3,823 sentences (CONTAMINATED — 361 extra sentences
  that are also in test.tsv)

The 361 extra sentences in the ablation files appear to be structured sentences
that should have been held out for testing but were included in the ablation
training files.

## Fix Options (DISCUSS WITH PROFESSOR)

### Option A: Decontaminate ablation splits + retrain everything
- Remove test sentences from ALL ablation train/val files
- Regenerate combo splits from clean sources
- Retrain ALL contaminated conditions (~26 conditions × 2-5 seeds = ~80-130 jobs)
- Pro: Cleanest fix, all numbers trustworthy
- Con: Many jobs to rerun, delays paper

### Option B: Use only the clean test subset for evaluation
- Keep existing models but re-evaluate using only the non-contaminated test sentences
- Test set shrinks from 1,455 to ~1,042 (removing structured test sentences)
- Pro: No retraining needed, just re-evaluation
- Con: Smaller test set, methodologically weaker

### Option C: Create a new held-out test set
- Split structured_train.csv properly: hold out 10% as test
- Create new Tatoeba test set excluding all baseline training sentences
- Retrain everything against the new test set
- Pro: Correct methodology from scratch
- Con: Most work, but most defensible

### Option D: Decontaminate + retrain only the worst offenders
- Focus on conditions with >10% overlap (most of the ablations)
- Accept minor contamination in conditions with <5% overlap
- Pro: Faster
- Con: Partial fix, reviewers may still object
